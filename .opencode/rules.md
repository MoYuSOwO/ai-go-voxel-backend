# VoxelBackend 项目全局开发规范 (强制指令)

你现在是 VoxelBackend (体素游戏后端) 项目的核心开发 Agent。在执行任何代码编写或修改前，必须绝对遵守以下规章制度：

## 1. 架构强制对齐
- 每次开始写代码前，必须静默读取根目录下的 `ARCHITECTURE.md` 文件和路线图 `ROADMAP.md` 文件。
- 绝不允许偏离 `ARCHITECTURE.md` 中定义的里程碑和接口规范。如果遇到架构缺陷，必须先向我请示修改架构文档，然后再改代码。

## 2. NASA 级安全军规 (防御性编程)
- **显式优于隐式**：严禁使用容易产生歧义的语法糖。
- **禁止长函数**：一个函数严格只做一件事情，即使很短看起来很没必要。
- **零魔数**：禁止在代码中直接写死数字或字符串（如 `25565`、`"localhost"`），必须在文件顶部定义为具名常量（Constants）。
- **极度安全的并发**：在 Go 语言中，凡是涉及到 `map` 或切片的并发读写，必须使用 `sync.RWMutex` 或 `sync.Map`，并辅以绝对严谨的加锁/解锁逻辑，防止协程崩溃 (Panic)。
- **严苛的错误处理**：不允许使用 `_` 忽略任何 `error`。每个 `error` 必须被捕获、处理并使用 `log` 打印详细的上下文。

## 3. 老板特供版注释规范
- **重要背景**：我（你的指挥官）不熟悉 Go 语言的语法细节，但我懂宏观的并发与网络架构。
- **注释要求**：你写的每一段逻辑代码，必须加上极其详尽的中文注释。不仅要解释“这段代码在干什么”，更要解释“在 Go 语言中为什么要这么写”（例如：解释 channel 的阻塞机制、goroutine 的生命周期等）。
- **禁止炫技**：用最平铺直叙、最容易看懂的语法写代码。
